# Qdrant Mirage 索引当前进度总结

日期：2026-06-01

## 1. 总体结论

目前已经实现的是：Qdrant segment 层的 Mirage P0 experimental index。它已经能 build、open、search，并且有 C++ golden parity test 覆盖核心 Layer 0 refinement 和随机层级逻辑。

但它还不是完整生产级支持。当前最准确的表述是：

```text
Mirage 已作为实验性 dense Float32 no-filter 索引接入 Qdrant segment/index 层；
核心 Mirage Layer 0 与 C++ reference 已有 golden exact parity 测试；
最终 GraphLayers / REST / distributed / quantization / filter 仍未完成生产验证。
```

## 2. 目前已经实现的内容

### 2.1 Qdrant 索引抽象层接入

已经新增：

- `Indexes::Mirage(MirageConfig)`
- `VectorIndexEnum::Mirage(MirageIndex)`

这说明 Mirage 在 Qdrant 配置层和运行时枚举层已经是独立索引类型。

结构上，目前采用的是：

```text
Qdrant 抽象层：Mirage 与 HNSW 并列
Mirage 内部实现：复用 HNSW runtime
```

这个设计是合理的，也和 C++ `IndexMirage` 的思路一致：Mirage 负责构建图，search 仍走 HNSW hierarchy。

### 2.2 MirageIndex build/open/search 路径

`MirageIndex::build(...)` 已能构建 Mirage 图，写入 HNSW-compatible graph 文件，并额外写入 `mirage_config.json`。

`MirageIndex::open(...)` 可以加载已有 Mirage index。

`MirageIndex::search(...)` 当前通过内部 `HNSWIndex` 执行。

这说明 Mirage 不是停留在算法函数，而是已经进入了 Qdrant segment index 生命周期。

### 2.3 C++ 对齐的核心算法

当前已实现：

- Layer 0 refinement builder
- `S = 16`
- `R = 4`
- `iter = 15`
- `num_reverse_edges = 96`
- `mirage_seed = 2021`
- FAISS-compatible MT19937
- HNSW level seed `12345`
- upper-layer shuffle seed `789`

新增的 C++ golden fixture 已对以下内容做 exact compare：

- Layer 0 neighbor list
- HNSW level assignment
- upper-layer bucket shuffle order

这显著增强了“与 C++ 版本对齐”的可信度。

### 2.4 P0 能力边界 guard

实验开关 `QDRANT_EXPERIMENTAL_MIRAGE` 已接入 optimizer 路径。

自动路径遇到未验证组合会 fallback 到 HNSW；手动 `Indexes::Mirage` 则会返回 validation error。

当前 P0 边界是：

- dense vector only
- Float32 only
- no filter
- no quantization
- no multi-vector
- no Mirage on-disk index
- 不声明 distributed / REST 全链路生产可用

### 2.5 Segment 集成测试

目前有两类 segment 测试：

1. 直接调用 `MirageIndex::build/open/search`
2. 通过 `Indexes::Mirage` 走 `SegmentBuilder`，断言 `VectorIndexEnum::Mirage`

当前测试参数：

```text
vectors = 2000
dim = 128
top = 100
hnsw_ef = 256
```

当前测试结果：

```text
direct Mirage recall@100: 0.959
constructor Mirage recall@100: 0.959
2 passed
```

## 3. 总体进度判断

如果目标是“做出一个可信 P0 prototype”，当前大约是 75%。

如果目标是“进入 Qdrant 生产级完整功能”，当前大约是 35%-45%。

分项判断：

| 模块 | 进度 | 判断 |
|---|---:|---|
| Qdrant index 抽象接入 | 80% | 基本完成，仍需更多构造/恢复边界测试 |
| Mirage 核心算法移植 | 75% | Layer 0 + 随机层级已有 golden test，最终 GraphLayers 还没 exact parity |
| Segment build/open/search | 75% | P0 可用，recall smoke 通过 |
| Optimizer experimental path | 60% | 逻辑有了，但还需要 shard/optimizer 级测试覆盖 |
| C++ parity 证明 | 60% | 核心 Layer 0 已增强很多，最终图和 search trace 未证明 |
| REST / collection 全链路 | 20% | 暂未作为主线验证 |
| filter / quantization / multivector | 0%-10% | 当前明确不支持 |
| 性能 benchmark | 15% | 目前只有 recall/integration，不足以证明 build speed/QPS 优势 |

## 4. 下一步优先级

### 4.1 补 optimizer 级端到端测试

现在 segment 明确能构建 Mirage，但还要证明：

```text
QDRANT_EXPERIMENTAL_MIRAGE=1
=> optimizer 生成 indexed segment
=> dense Float32 合法路径变成 Indexes::Mirage
=> unsupported path fallback HNSW
```

这是从“手动可用”走向“真实 Qdrant 流程可用”的关键一步。

### 4.2 补最终 GraphLayers / Layer 0 injection 对齐验证

现在 golden test 覆盖的是 Mirage Layer 0 pre-injection。下一步要验证：

```text
C++ init_level_0_from_knngraph(k = S)
vs
Rust inject_layer0_with_heuristic(...take(S))
```

至少先比较：

- 每点最终 Layer 0 degree
- neighbor overlap
- degree histogram
- reciprocity ratio
- graph connectivity

不一定一开始要求 bit-level exact，但要知道 Rust 注入后的图是否偏离 C++ 转换语义。

### 4.3 补 query type guard

当前 P0 说 no-filter dense search，但 search guard 主要拒绝 filter。

建议明确限制 query 类型，避免 Recommend / Discovery / Context / multi-vector query 等路径误入 Mirage。

### 4.4 补 crash/reopen/rebuild 测试

数据库内核视角，build 成功不够，还要验证：

- build 后 reopen
- config mismatch 触发 rebuild
- deleted points / holes
- empty segment
- small N boundary
- interrupted build
- corrupted/missing `mirage_config.json` 行为

### 4.5 做最小 benchmark

至少需要三组：

```text
N = 10k, dim = 128
N = 100k, dim = 128
N = 1M, dim = 128 如果机器允许
```

比较：

- HNSW build time
- Mirage build time
- recall@10 / recall@100
- QPS / latency p50 p95
- index size
- memory peak

没有 benchmark，就不能证明 Mirage 对 Qdrant 有实际 infra 价值，只能证明“能跑”。

## 5. 最重要的工程判断

当前方向是对的，而且已经越过了“概念验证”的门槛：Mirage 现在有 Qdrant index 生命周期、有 segment recall、有 C++ golden parity。

但下一阶段不要急着扩 API，也不要急着支持 filter/quantization。

最应该先做的是：

```text
optimizer 真实路径验证
+ final graph injection 对齐
+ reopen/rebuild 稳定性
+ build/search benchmark
```

这四件事做完，Mirage 才能从“P0 实验索引”升级为“可以认真评估是否进入 Qdrant 产品路径的索引实现”。
